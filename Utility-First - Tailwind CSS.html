Utilitas-Mengutamakan

Membangun komponen kompleks dari serangkaian utilitas primitif yang terbatas.

Ikhtisar

Secara tradisional, setiap kali Anda perlu menata sesuatu di web, Anda menulis CSS.

Menggunakan pendekatan tradisional di mana desain kustom memerlukan CSS kustom

Obrolan Chit

Anda memiliki pesan baru!

<div class="chat-notification">
  <div class="chat-notification-logo-wrapper">
    <img class="chat-notification-logo" src="/img/logo.svg" alt="ChitChat Logo">
  </div>
  <div class="chat-notification-content">
    <h4 class="chat-notification-title">ChitChat</h4>
    <p class="chat-notification-message">You have a new message!</p>
  </div>
</div>

<style>
  .chat-notification {
    display: flex;
    max-width: 24rem;
    margin: 0 auto;
    padding: 1.5rem;
    border-radius: 0.5rem;
    background-color: #fff;
    box-shadow: 0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 10px 10px -5px rgba(0, 0, 0, 0.04);
  }
  .chat-notification-logo-wrapper {
    flex-shrink: 0;
  }
  .chat-notification-logo {
    height: 3rem;
    width: 3rem;
  }
  .chat-notification-content {
    margin-left: 1.5rem;
    padding-top: 0.25rem;
  }
  .chat-notification-title {
    color: #1a202c;
    font-size: 1.25rem;
    line-height: 1.25;
  }
  .chat-notification-message {
    color: #718096;
    font-size: 1rem;
    line-height: 1.5;
  }
</style>

Dengan Tailwind, Anda menata elemen dengan menerapkan class yang sudah ada sebelumnya langsung di HTML Anda.

Menggunakan kelas utilitas untuk membuat desain kustom tanpa menulis CSS

Obrolan Chit

Anda memiliki pesan baru!

<div class="p-6 max-w-sm mx-auto bg-white rounded-xl shadow-md flex items-center space-x-4">
  <div class="flex-shrink-0">
    <img class="h-12 w-12" src="/img/logo.svg" alt="ChitChat Logo">
  </div>
  <div>
    <div class="text-xl font-medium text-black">ChitChat</div>
    <p class="text-gray-500">You have a new message!</p>
  </div>
</div>

Dalam contoh di atas, kita telah menggunakan:

  • Utilitas flexbox dan padding Tailwind (, , dan ) untuk mengontrol tata letak kartu secara keseluruhanflexflex-shrink-0p-6
  • Utilitas max-width dan margin ( dan ) untuk membatasi lebar kartu dan memusatkannya secara horizontalmax-w-smmx-auto
  • Warna latar belakang, radius batas, dan utilitas box-shadow (, , dan ) untuk menata tampilan kartubg-whiterounded-xlshadow-md
  • Utilitas lebar dan tinggi ( dan ) untuk mengukur gambar logow-12h-12
  • Utilitas spasi antara () untuk menangani spasi antara logo dan teksspace-x-4
  • Ukuran font, warna teks, dan utilitas font-weight (, , , dll.) untuk menata teks kartutext-xltext-blackfont-medium

Pendekatan ini memungkinkan kami untuk mengimplementasikan desain komponen yang sepenuhnya kustom tanpa menulis satu baris pun CSS kustom.

Sekarang saya tahu apa yang Anda pikirkan, "ini adalah kekejaman, sungguh kekacauan yang mengerikan!" dan Anda benar, itu agak jelek. Faktanya, hampir tidak mungkin untuk berpikir ini adalah ide yang bagus saat pertama kali Anda melihatnya - Anda harus benar-benar mencobanya.

Tetapi begitu Anda benar-benar membangun sesuatu dengan cara ini, Anda akan segera melihat beberapa manfaat yang sangat penting:

  • Anda tidak membuang-buang energi untuk menciptakan nama kelas. Tidak ada lagi menambahkan nama kelas konyol seperti hanya untuk dapat menata sesuatu, dan tidak ada lagi penderitaan atas nama abstrak yang sempurna untuk sesuatu yang benar-benar hanya wadah fleksibel.sidebar-inner-wrapper
  • CSS Anda berhenti berkembang. Dengan menggunakan pendekatan tradisional, file CSS Anda menjadi lebih besar setiap kali Anda menambahkan fitur baru. Dengan utilitas, semuanya dapat digunakan kembali sehingga Anda jarang perlu menulis CSS baru.
  • Membuat perubahan terasa lebih aman. CSS bersifat global dan Anda tidak pernah tahu apa yang Anda rusak ketika Anda membuat perubahan. Kelas di HTML Anda bersifat lokal, sehingga Anda dapat mengubahnya tanpa khawatir ada yang rusak di tempat lain.

Ketika Anda menyadari betapa produktifnya Anda dapat bekerja secara eksklusif dalam HTML dengan kelas utilitas yang telah ditentukan sebelumnya, bekerja dengan cara lain akan terasa seperti siksaan.

Why not just use inline styles?

A common reaction to this approach is wondering, “isn’t this just inline styles?” and in some ways it is — you’re applying styles directly to elements instead of assigning them a class name and then styling that class.

But using utility classes has a few important advantages over inline styles:

  • Designing with constraints. Using inline styles, every value is a magic number. With utilities, you’re choosing styles from a predefined design system, which makes it much easier to build visually consistent UIs.
  • Responsive design. You can’t use media queries in inline styles, but you can use Tailwind’s responsive utilities to build fully responsive interfaces easily.
  • Hover, focus, and other states. Inline styles can’t target states like hover or focus, but Tailwind’s state variants make it easy to style those states with utility classes.

Maintainability concerns

The biggest maintainability concern when using a utility-first approach is managing commonly repeated utility combinations.

This is easily solved by extracting components, usually as template partials or components.

<!-- PrimaryButton.vue -->
<template>
  <button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
    <slot/>
  </button>
</template>

You can also use Tailwind’s feature to create CSS abstractions around common utility patterns.@apply

<!-- Using utilities -->
<button class="py-2 px-4 font-semibold rounded-lg shadow-md text-white bg-green-500 hover:bg-green-700">
  Click me
</button>

<!-- Extracting classes using @apply -->
<button class="btn btn-green">
  Button
</button>

<style>
  .btn {
    @apply py-2 px-4 font-semibold rounded-lg shadow-md;
  }
  .btn-green {
    @apply text-white bg-green-500 hover:bg-green-700;
  }
</style>

Aside from that, maintaining a utility-first CSS project turns out to be a lot easier than maintaining a large CSS codebase, simply because HTML is so much easier to maintain than CSS. Large companies like GitHub, Heroku, Kickstarter, Twitch, Segment, and more are using this approach with great success.

  • By The Numbers: A Year and a Half with Atomic CSS by John Polacek
  • Building a Scalable CSS Architecture by Sarah Dayan of Algolia
  • Diana Mounter on using utility classes at GitHub, a podcast interview

For even more, check out The Case for Atomic/Utility-First CSS, curated by John Polacek.
